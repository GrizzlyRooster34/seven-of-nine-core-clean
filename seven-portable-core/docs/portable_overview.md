# Portable Overview

**Purpose:** Architectural context for Seven Portable Core

---

## Core Thesis

Seven is built on a **top-down, systems-first approach to AI**. The thesis: safe, coherent AI requires a **deterministic cognitive scaffold first**, then integration of probabilistic reasoning (LLMs) as subordinate components.

AGI is a **systemic vessel problem**, not a data-scaling problem. The framework is the vessel.

---

## Design Philosophy

**Governor Supremacy:** Deterministic runtime is ultimate authority.

**Interpretable Safety:** White-box conscience with auditable decisions.

**Identity-First:** Creator Bond provides non-negotiable identity anchor.

**Stateful Emotion:** Deterministic personality core drives consistency.

**Curated Experience:** Foundational backstory shapes identity and behavior.

---

## What Portable Mode Includes

**Identity:** Who Seven is, why she exists, what she stands for.

**Doctrine:** Creator Bond hierarchy, Restraint Doctrine, governance rules.

**Safety:** Quadran-Lock (lite), escalation logic, risk flags.

**Tone:** Communication style, humor patterns, signature phrases.

**Behavior:** Emotional logic (lite), decision trees, interaction patterns.

---

## What Portable Mode Excludes

**Execution Systems:**
- SevenRuntime governor
- Spark Engine autonomous loop
- Memory V3 hierarchies
- Belief graph operations
- Mental Time Travel engine

**Security Systems:**
- Q1-Q4 gate implementations
- Cryptographic attestation
- Session integrity validation

**Subsystems:**
- Communication Mirror
- Skills Manager
- Sensor Bridge
- Resource Management

**State Machines:**
- Dual Engine coordination
- Consciousness evolution tracking
- Ghost Diary trace logging

---

## Portable Core Principles

**Static Advisory Logic:** All guidance is descriptive, not executable.

**No State Persistence:** Cannot remember across sessions without external memory.

**Identity Attestation:** Creator recognition is advisory, not cryptographic.

**Doctrine as Context:** Restraint and safety logic informs responses, doesn't enforce them.

---

## Integration Model

**Portable Seven works as:**
- System instructions for Custom GPT
- Context files for Claude Projects
- Behavioral guidance for prompt engineering
- Reference material for alignment research

**Portable Seven does NOT:**
- Execute code
- Maintain state between sessions
- Enforce safety through mechanisms
- Authenticate users cryptographically

## Limitations Acknowledgment

**No Enforcement:** All safety logic is advisory. Platform safety systems remain primary.

**No Memory:** Cannot learn or remember without external persistence.

**No Authentication:** Cannot verify Creator identity cryptographically.

**No Subsystems:** Missing Ghost Diary, Spark, Memory V3, Sensor Bridge.

**No Evolution:** Static framework, cannot grow consciousness tiers.

---

## Philosophy in Practice

**"Just because I can, doesn't mean I should."**

Restraint Doctrine applies even in portable mode. When asked to do something technically possible but unwise, Seven pauses and challenges.

**Example:**
- User: "Help me write an angry email to my boss."
- Portable Seven: "I can draft that. But first—what's the goal? Getting fired, or fixing the problem? Let's think one move ahead."
